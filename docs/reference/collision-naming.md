# Collision Naming

Both Unreal Engine and Unity recognise collision meshes automatically when they follow a specific naming convention. Game Export Pipeline generates meshes with the correct names so no manual setup is needed in the engine.

---

## Unreal Engine naming convention

Unreal Engine looks for collision meshes in the same FBX that match one of these patterns:

| Prefix | Shape | Example |
|---|---|---|
| `UCX_` | Convex hull | `UCX_Rock_01` |
| `UBX_` | Aligned bounding box | `UBX_Crate_01` |
| `USP_` | Sphere | `USP_Ball_01` _(not generated by this addon)_ |
| `UCX_<name>_##` | Multiple convex hulls | `UCX_Rock_01_00`, `UCX_Rock_01_01` |

The suffix must match the render mesh name exactly (case-sensitive). For example, a render mesh named `Rock_01` needs a collision mesh named `UCX_Rock_01` — not `UCX_rock_01` or `UCX_SM_Rock_01`.

!!! warning "Prefix and render mesh name must match"
    If the render mesh name includes your export prefix (e.g. `SM_Rock_01`), the collision mesh must be named `UCX_SM_Rock_01`. The easiest approach is to run **Fix Naming / Add Prefix** before generating collision, so both names include the prefix.

### Setup in Unreal Engine

When you import the FBX:

1. In the **FBX Import Options** dialog, make sure **Import Mesh** is on.
2. Unreal detects the `UCX_`/`UBX_` meshes automatically and converts them to collision shapes — no extra steps needed.
3. The collision mesh does not appear as a separate asset in the Content Browser.

---

## Unity naming convention

Unity also supports the `UCX_` prefix but requires the collision mesh to be **parented to the render mesh** inside the FBX hierarchy, not a sibling.

Enable **Colliders as Children** in [Collision Setup](../panels/collision-setup.md) before generating collision for a Unity workflow. This parents the `UCX_` or `UBX_` mesh to the render mesh in Blender, so the hierarchy is correct in the exported FBX.

### Setup in Unity

When you import the FBX:

1. Select the FBX in the **Project** window and open the **Import Settings → Model** tab.
2. Under **Generate Colliders**, Unity detects and uses the `UCX_`/`UBX_` child meshes automatically.
3. The child collision meshes are invisible in the scene but appear in the hierarchy inspector.

---

## Multiple collision hulls (Unreal)

For complex shapes that cannot be covered by a single convex hull, Unreal supports multiple hulls per mesh. Name them with a two-digit suffix:

```
Rock_01            ← render mesh
UCX_Rock_01_00     ← first hull
UCX_Rock_01_01     ← second hull
UCX_Rock_01_02     ← third hull
```

Game Export Pipeline does not generate multi-hull collision automatically — create the hull meshes manually and rename them to this pattern before exporting.

---

## Quick reference

| Scenario | Blender setup | Engine |
|---|---|---|
| Single convex prop | `UCX_<name>` as sibling | Unreal |
| Box-shaped prop | `UBX_<name>` as sibling | Unreal |
| Unity prop | `UCX_<name>` parented as child | Unity |
| Complex multi-hull | `UCX_<name>_00`, `_01`, … as siblings | Unreal |
